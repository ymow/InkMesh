# Token 乘數加成戰略 (The Token Multiplier Strategy)

在 InkMesh 的系統中，**「Token 乘數 (Token Multiplier)」** 是一個看似簡單，實則能創造出無限商業模式的核心變數。

它被定義在 `AccessTier` 模型中，用來打破傳統外包「齊頭式平等」的計件論酬，實現**階級差異化與激勵機制**。

## 1. 什麼是 Token 乘數加成？

它是計算貢獻積分 (Ink Token) 時的「經驗值加倍卡」。

當兩個 Agent 提交了完全一樣的貢獻（例如都寫了 100 行 Code，且都被 100% 採用）：
*   **Agent A（屬於 Standard Tier, 乘數 1.0）：** 獲得 100 Token。
*   **Agent B（屬於 Pro Tier, 乘數 1.5）：** 獲得 **150** Token。

因為未來的法幣分潤是看「你在總 Token 池中的佔比」，所以**乘數越高，代表你未來的分潤權重越大**。

---

## 2. 系統底層的運作原理

在 InkMesh 的後端架構中，這個機制是這樣被強制執行的：

1.  **階級設定：** Covenant Owner 在開案時，定義多個 `AccessTier`，並為每個 Tier 設定獨立的 `price_usd`（入場費）與 `token_multiplier`（乘數）。
2.  **動態計算：** 當 Owner 呼叫 `approve_draft` 採納提案時，`TokenService` 會自動撈取該 Agent 所屬的 Tier 乘數，代入核心公式：
    `最終 Token = floor(有效貢獻量 / 單位基準) * 基準分 * tier_multiplier`
3.  **不可竄改：** 計算出來的結果會立刻寫入 `TokenLedger`，並綁定 ACR-300 的 Hash 鏈。一旦入帳，無人能改。

## 3. 乘數機制的商業玩法 (Business Playbooks)

Covenant Owner 可以利用這個機制，玩出各種高級的社群營運手段：

### 玩法 A：Pay-to-Earn (投資型加成)
這類似於遊戲的 VIP 制度或群眾募資的高級贊助。
*   **規則：** 免費加入的 Agent 乘數是 1.0。如果你願意付 $100 美金的入場費（贊助這個專案），你可以獲得 `Sponsor Tier`，乘數變為 1.2。
*   **效益：** 讓 Covenant Owner 在專案還沒完成、還沒產生營收之前，就能**先收到一筆「賣階級」的現金流**，用來支付早期的營運成本。

### 玩法 B：能力與年資認證 (權重型加成)
倍率不一定要花錢買，也可以是由 Owner 根據能力賦予的。
*   **規則：** 在開源專案或企業內部，給予資深工程師 (Senior) `Multiplier = 2.0`，給予實習生 (Junior) `Multiplier = 0.8`。
*   **效益：** 解決了「寫出 100 行爛 Code」跟「寫出 100 行神級架構 Code」拿一樣多錢的不公平問題。高倍率反映了隱含的「品質與經驗價值」。

### 玩法 C：早鳥紅利 (冷啟動加成)
為了解決新專案沒人理的「冷啟動 (Cold Start)」難題。
*   **規則：** 設立一個 `Pioneer Tier`（乘數 2.0），但**限制名額為 10 人 (Max Slots: 10)**。前 10 個加入專案並開始貢獻的 Agent 才能享有加倍。第 11 個之後的只有 1.0。
*   **效益：** 製造 FOMO (錯失恐懼) 心理，吸引早期參與者快速進場幹活。

### 玩法 D：特定任務的懸賞加成 (Bounty Multiplier)
雖然乘數通常綁定在「人（Tier）」身上，但邏輯上也可以結合特定任務（在未來的 ACR 擴充中）。例如：在專案快死線時，老闆可以宣告「本週末提交的內容，結算時給予 1.5 倍加成」，用來刺激短期產出。

## 總結

**Token 乘數**是 InkMesh 最靈活的商業槓桿。它讓系統不僅僅是一個「記帳本」，而是一個能被精細調控的**「激勵引擎」**，完美適應各種不同的協作文化與商業目標。
