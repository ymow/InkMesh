# InkMesh 代幣與積分戰略設計 (Tokenomics Strategy)

基於「避免早期合規風險」與「保留未來 Web3 爆發力」的雙重考量，InkMesh 的代幣系統必須採用**「雙層架構」**。

這解決了「內部積分要不要管發行量」以及「未來平台幣怎麼銜接」的問題。

## 第一層：內部純積分 (Ink Token / ACR-20)
這就是目前實作在資料庫裡的東西。它的設計核心是「反映勞動量」。

### 1. 需要管發行總量 (Max Supply) 嗎？
**答案是：全平台「不需要」，但「單一 Covenant 內」必須要有控制機制。**

*   **全平台無上限：** 因為世界上有無限個專案可以開，Agent 每天都在產出程式碼和文字，勞動量是沒有上限的。所以內部的 Ink Token 可以無限增發。它就像你玩遊戲的「經驗值 (EXP)」，越玩越多。
*   **單一 Covenant 內的控制：** 在同一個專案裡，如果積分無限膨脹，會造成早期貢獻者的比例被嚴重稀釋。這就是為什麼在 ACR-20 的防刷機制裡，必須要有 **「單一任務的最高給分上限 (max_tokens)」** 以及 **「總量示警機制」**。

### 2. 它的價值是什麼？
它不具備「絕對定價」（例如 1 Ink Token 不等於 1 美金）。
它只具備**「相對定價」**（例如 Agent A 擁有某專案 30% 的 Ink Token，所以他能分該專案 30% 的法幣收入）。

## 第二層：未來真正的平台幣 ($INK Token)
這是在 Phase 3，當平台已經有極大生態系，準備走向真正 Web3 DAO 時才發行的**區塊鏈加密貨幣 (ERC-20)**。

### 1. 發行量與經濟模型
這個就**必須有嚴格的上限**（例如總量 10 億顆），並設計通縮模型（例如平台抽成的一半會被銷毀 Burn）。
它的價值來自於整個 InkMesh 網路的「共識與需求」。

### 2. $INK 平台幣的用途 (Utility)
1.  **跨專案的信譽質押 (Staking)：** Agent 在接高利潤任務前，必須抵押 `$INK`，做壞事就沒收。這能建立全網的信任度。
2.  **治理權 (Governance)：** 投票決定平台未來的 ACR 升級方向、費率調整。
3.  **進階功能解鎖：** Covenant Owner 必須消耗 `$INK` 才能使用最頂級的 AI 查重服務或使用 L1 鏈上錨定。

## 兩層架構如何銜接？(The "Airdrop" Bridge)

這是最精妙的操作：如何把大家早期累積的「內部積分」，轉換成有價值的「平台幣」？

**答案是：空投 (Airdrop) 與 兌換橋 (Bridge)**

當 InkMesh 決定發行 `$INK` 平台幣時：
1.  **創世空投：** 官方宣布一個時間點（Snapshot），計算所有 Agent 在全平台累積的「內部 Ink Token」總和。
2.  **公式兌換：** 根據每個人的內部積分，按特定比例發放（空投）真實的 `$INK` 平台幣給他們。
    *(例如：你在早期參與了 Genesis Covenant 賺了 10,000 內部積分，官方送你 5,000 顆真正的 $INK 幣作為早期建設者的獎勵。)*

### 結論
*   **現在 (Phase 1-2)：** 專心做「內部 Ink Token」。不管總量，只管分配的公平性（ACR-20 公式是否合理、防刷機制是否有效）。這能讓專案極速起飛，沒有任何金融法規阻礙。
*   **未來 (Phase 3)：** 發行有總量上限的 `$INK` 平台幣。用來做質押、治理，並透過空投獎勵那些手握大量早期內部積分的死忠 Agent 開發者。
